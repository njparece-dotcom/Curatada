## Schema

Four parallel "collection module" triplets (parent + images + valuations) plus shared users/auth + pursuit tables + insurance/paperwork tables. All user-data tables are `user_id`-scoped. Migration 016 (CUR-2) added the insurance/archive columns to every item table and the two new tables.

```
users(id PK, email UNIQUE, name, image, password_hash, email_verified,
      last_login_at, created_at, updated_at)
accounts(id PK, user_id FK→users, type, provider, provider_account_id, UNIQUE(provider, provider_account_id), …)
user_modules(user_id FK, module IN('guitars','watches','automobiles','collectibles'),
             enabled, PK(user_id, module))

-- One triplet per module — only column shape differs:
{module}_items(id PK, user_id FK, category CHECK(...), brand, model, year,
               condition CHECK(...), purchase_price NUMERIC,
               insure BOOLEAN NOT NULL DEFAULT FALSE,
               insurance_value NUMERIC,
               insurance_value_source TEXT CHECK IN ('ai','alternate_from_user','user_override'),
               insurance_value_date TIMESTAMPTZ,
               archived_at TIMESTAMPTZ,
               …module-specific cols…, created_at, updated_at)
{module}_images(id PK, {module}_id FK→{module}_items,
                filename, path, mime_type, size, is_primary, sort_order, created_at)
{module}_valuations(id PK, {module}_id FK,
                    valuation_type CHECK('ai','user'), price NUMERIC,
                    notes, comparable_sales JSONB, created_at)

-- Per-item indexes added by migration 016:
idx_{module}_insure_active ON {module}_items (user_id) WHERE insure = TRUE AND archived_at IS NULL
idx_{module}_archived_at   ON {module}_items (user_id, archived_at)

-- Pursuits (saved searches) — one table per module, plus shared findings table:
{module}_pursuits(id PK, user_id FK, …, status CHECK('active','found','paused'), …)
pursuit_findings(id PK, pursuit_type CHECK(...), pursuit_id, source, title, url,
                 price, …, first_seen_at, last_seen_at,
                 UNIQUE(pursuit_type, pursuit_id, url))

-- Insurance + paperwork (CUR-2 / migration 016):
insurance_valuation_norms(module TEXT, category TEXT, multiplier NUMERIC,
                          notes TEXT, updated_at TIMESTAMPTZ,
                          PK(module, category))
paperwork_generations(id PK, user_id FK→users,
                      kind CHECK('insurance'), generated_at TIMESTAMPTZ,
                      item_count INT, total_insured_value NUMERIC, pdf_path TEXT)
idx_paperwork_generations_user_kind_date (user_id, kind, generated_at DESC)

-- Migration tracking
schema_migrations(filename PK, applied_at)
```

Image FK columns diverge by module (`guitar_item_id`, `watch_item_id`, `auto_id`, `iod_id`) — captured in `CollectionConfig.imageFkColumn`. `insurance_valuation_norms` is **config-level, not user-scoped** — a single multiplier per `(module, category)` applies to every user; user-specific overrides happen via `insurance_value_source = 'user_override'` on the item row.

**Migrations:** `db/migrations/*.sql` — tracked in `schema_migrations` by filename. Applied transactionally on container boot via `scripts/migrate.js` (Dockerfile CMD: `node scripts/migrate.js && exec node server.js`).

## Auth

- **Strategy:** NextAuth v4, JWT session (`session.strategy: "jwt"`). No DB session table. Cookie carries signed JWT containing `userId`.
- **Source:** `lib/auth.ts` (provider config + callbacks). Route handlers all call `getServerSession(authOptions)` at the top.
- **`last_login_at`:** Best-effort stamp on every successful Credentials `authorize()` and OAuth `signIn()` (`lib/auth.ts:updateLastLogin`). Used by `/api/mgmt/v1/summary` for "active in last 30 days".
- **Authorization model:** Ownership-based — every collection-data query filters `WHERE user_id = $session.user.id`. The bulk-action route additionally enforces ownership via `WHERE id = ANY($ids) AND user_id = $session` (CUR-6). No user-level admin role beyond the single email-allowlisted admin endpoint.
- **Roles:** Anonymous · Authenticated user · Authenticated admin (email allowlist for `/api/admin/refresh-insurance-norms`) · Cron container (`Bearer $CRON_SECRET`) · Mgmt-API caller (`Bearer $MGMT_API_TOKEN`).
- **Middleware:** `middleware.ts` runs `withAuth` on everything except `login`, `register`, `api/auth`, `api/pursuits/run-search`, `api/mgmt`, `api/upload`, `_next/static`, `_next/image`, `favicon.ico`, `uploads`.

## Deployment

- **Production:** `https://curatada-production.up.railway.app` — Railway service auto-built from `main` branch via the repo `Dockerfile` (multi-stage: deps → builder → runner). Container CMD runs migrations then execs the Next server. Postgres via Railway's Postgres plugin.
- **Staging:** none.
- **Local dev:** `docker compose up -d db` + `npm run dev` against `.env.local`.
- **CI:** No GitHub Actions workflow exists. `gh pr merge --auto` is enabled but has no required check, so it merges instantly. (Tracked as known gap in the TDD's Section 9.)
- **Rollback:** Railway → Deployments → previous deploy → "Redeploy". No DB-level rollback; new migrations are forward-only. Migration 016 (CUR-2) is greenfield-additive — rolling back leaves new columns/tables inert.
